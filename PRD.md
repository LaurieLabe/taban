# 她伴 · 产品需求文档（PRD）

> 去性缘化的女性搭子匹配平台
> 版本：v3.2 | 2026-07-20

---

## 一、产品定位

**她伴**是一个去性缘化的女性搭子匹配平台。解决成年女性离开学校后，同性友缘关系因缺乏制度化的匹配工具而被性缘关系侵蚀和替代的困境。我们不是"帮女性找对象"，而是帮女性搭建专属的友缘共同体。

**一句话定义：** 让女性轻松找到非恋爱导向的、精神同频的女性伙伴。

**核心价值观：**

- **去性缘化：** 纯女性空间，不为恋爱关系提供任何脚本和暗示
- **目的性克制：** 不让"人"像商品一样被推送，用"事"作为轻盈的缓冲区
- **连接而非撮合：** 关系自然生长，不被算法强行驱动
- **温和的节奏感：** 每日发起限制不是防骚扰，而是提醒——搭子不急于一天

---

## 二、目标用户

**核心用户画像：** 20-40岁城市女性，渴望在伴侣、同事之外拓展轻松、深度的同性友谊。

**细分场景：**

| 场景 | 痛点 |
|------|------|
| 异乡独居者 | 在陌生城市缺乏社交网络 |
| 远程工作者 | 办公场景没有同事社交 |
| 新手妈妈 | 社交圈因育儿而收缩 |
| 稳定伴侣关系中的女性 | 社交网络逐渐收缩至"伴侣"单一节点 |

## 三、核心功能

### 架构总览

```
注册 → 显性问卷（6维度勾选）→ AI隐性对话（3-5轮）→ 生成友缘画像
                                                         │
                                    用户创建人格卡片（文字/图片）
                                                         │
                                    进入主页 ← 大门打开动画
                                      │
              ┌───────────────────────┼───────────────────────┐
              │           │           │           │           │
            探趣         觅友         ＋         消息         我
         (首页)    (人格卡片)    (发布事件)   (聊天列表)   (个人主页)
```

### 底部导航栏

| 标签 | 功能 |
|------|------|
| 探趣 | 首页，浏览和推荐搭子事件 |
| 觅友 | 浏览推荐的人格卡片，小红书卡片流风格 |
| ＋ | 发布搭子事件（醒目圆形按钮） |
| 消息 | 聊天列表，小红书消息界面风格 |
| 我 | 个人信息、标签展示、人格卡片管理 |

---

### 模块一：匹配"事"——首页

**定位：** 首页，作为"目的性克制"的缓冲区。用户打开 App 先看到的是附近的活动事件，而非一排排的人。

**功能：**

1. **事件流展示**
   - 以卡片形式展示附近/推荐的搭子事件
   - 每个事件卡片包含：活动标题、时间、地点、发起人简要信息、招募人数
   - 始终展示推荐内容，不做空状态（即使匹配度低也有推荐，类似美团综合排序）

2. **排序切换**
   - 综合推荐排序（默认）
   - 距离优先排序
   - 事件匹配度排序

3. **事件详情**
   - 点击卡片进入详情页
   - 展示完整事件信息 + 发起人显性标签 + 发起人人格卡片
   - "申请加入"按钮

4. **发布事件**
   - 入口：底部"＋"按钮
   - 填写事件标题、时间、地点、主题、人数限制
   - 预设女性向活动模板（拼豆、散步游湖、逛书店、手工、甜品、看展等）
   - 同时保留大众化活动选项（羽毛球、徒步、骑行、读书会等）
   - 性别限制：纯女性

---

### 模块二：匹配"人"——人格卡片

**定位：** 简朴克制，不让"人"像商品一样被展示。卡片排列稀疏，留白充足。

**功能：**

1. **人格卡片流**
   - 小红书笔记封面风格：一张卡片封面图 + 简要文字
   - 不展示精修自拍，展示用户选择的内容形式（文字/图片）
   - 基于用户画像的 AI 推荐排序

2. **卡片详情**
   - 点击卡片进入详情页
   - 展示：完整自我介绍、用户上传的图片、显性标签
   - "想认识"按钮

3. **单向发起机制**
   - 一方点击"想认识"即可发送一条消息
   - 对方回复后，双方建立对话
   - 每日发起上限：5次（温和提醒，非防骚扰）

---

### 模块三：消息

**功能：**

- 聊天列表，小红书消息界面风格
- 来自"觅友"的对话请求
- 来自"探趣"的申请加入通知
- 事件审批：发起人可查看申请列表并通过/拒绝
- 聊天界面：文本对话

---

### 模块四：我

**功能：**

- 个人基本信息
- 显性标签展示（6 维度）
- 人格卡片管理（编辑文字/图片）
- 设置：距离范围、通知偏好

## 四、匹配算法

### 双层匹配体系

| 层级 | 性质 | 来源 | 可见性 | 权重 |
|------|------|------|--------|------|
| 显性标签 | 客观/自知 | 用户主动勾选 | ✅ 可见 | 60% |
| 隐性量表 | 主观/潜在 | AI 对话推断 | ❌ 不可见 | 40% |

### 显性标签（6 维度）

| 维度 | 类型 | 说明 |
|------|------|------|
| 人格气质 | 单选 | MBTI 类型、内向/外向 |
| 兴趣活动 | 多选 | 徒步、看展、手工、逛书店、羽毛球、骑行、拼豆、学乐器、散步游湖、甜品探店、读书会……（女性向 + 大众化） |
| 聊天话题 | 多选 | 心理成长、文学、玄学、商业八卦、电影、音乐、旅行、美食…… |
| 消费观 | 组合 | 支付方式（AA制/轮流请客） + 消费风格（极简主义/品质享受），组合为"AA-品质享受"等 |
| 生活节奏 | 组合 | 作息（早起鸟/夜猫子） + 时间偏好（周末党/工作日自由），组合为"夜猫子-周末党"等 |
| 距离范围 | 单选 | 不限 / 同城 / 同区(县) / 10km以内 / 5km以内 |

### 隐性量表（5 维度，AI 推断，不可见）

| 维度 | 光谱 | AI 推断线索 |
|------|------|-------------|
| 情感共鸣度 | 高共情 ↔ 偏理性 | 对话中感受词频率、是否主动关心他人 |
| 主体性 | 主动发起型 ↔ 随性跟随型 | "我想"vs"都可以"的语言模式 |
| 社交能量 | 高能量输出 ↔ 需要空间 | 对话节奏、社交频率偏好 |
| 敏感度 | 高敏感 ↔ 钝感力强 | 细节关注度、情绪词汇丰富度 |
| 开放度 | 探索型 ↔ 安稳型 | 新体验态度、风险偏好表述 |

### 匹配公式

```
匹配得分 = 显性维度匹配度 × 0.6 + 隐性维度匹配度 × 0.4
```

## 五、用户完整流程

### 5.1 注册与画像构建

```
Step 1: 进入她伴
    │
Step 2: 注册（昵称 + 密码 + 出生年月日）
    │
Step 3: 显性问卷（6个页面）
    │   人格气质 → 兴趣活动 → 聊天话题 → 消费观 → 生活节奏 → 距离范围
    │   每页单独呈现，引领式问卷，不拥挤
    │
Step 4: AI 小助手弹出
    │   "让我作为你的第一个搭子，我们先来简单聊聊吧~"
    │   3-5轮自然对话，轻松有温度
    │   AI 分析对话，生成隐性量表（内部存储，不可见）
    │
Step 5: 生成友缘画像
    │   显性标签（用户确认可微调） + 隐性量表（AI推断，不可见）
    │
Step 6: 创建人格卡片
    │   选择形式：文字 / 图片
    │   填写自我介绍 & 上传图片
    │
Step 7: "欢迎进入她伴"
    │   大门打开动画，穿越到使用页面
    │   进入"事"（首页）
```

### 5.2 日常使用

```
主页（底部导航）
    │
    ├── 探趣（首页）：浏览事件 → 排序切换 → 查看详情 → 申请加入
    │
    ├── 觅友：浏览人格卡片 → 点击详情 → 想认识 → 发送消息
    │
    ├── ＋：发布搭子事件
    │
    ├── 消息：查看聊天/通知 → 回复消息
    │
    └── 我：查看/编辑个人信息、标签、人格卡片
```

## 六、AI 功能

> AI 在产品中承担推测分析和安全守护的角色。匹配排序的具体算法逻辑见第四章。

| AI 能力 | 说明 |
|---------|------|
| 隐性量表推断 | 通过自然对话分析用户语言模式，推断潜在特质维度（情感共鸣度、主体性、社交能量、敏感度、开放度） |
| 搭子推荐 | 基于显性标签 + 隐性量表的加权计算，为用户推荐匹配度高的潜在友缘 |
| 事件推荐 | 基于用户画像，推荐匹配度高的附近搭子事件 |
| AI 安全网 | 语义分析和异常行为监测，守护纯女性空间的安全 |

产品需要以下技术能力支撑：

## 八、UI 设计规范

### 8.1 视觉风格

- **整体调性：** 温暖、轻盈、克制、有呼吸感
- **配色：** 柔和暖色调为主，避免冷峻科技感
- **卡片风格：** 大圆角、柔和阴影、留白充足
- **"人"页面：** 特别克制，卡片排列稀疏，避免商品化展示

### 8.2 底部导航

- 5 栏等宽分布：事 / 人 / ＋ / 消息 / 我
- 当前选中页高亮
- 中间"＋"按钮视觉突出（主色填充圆形）

### 8.3 移动端适配

- 页面宽度：375-414px（模拟手机尺寸）
- 所有交互适配触屏操作
- 字体大小适配移动端阅读
